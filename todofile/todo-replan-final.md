# 重新规划功能待办清单

## 核心需求（严格按照用户文档）

### 1. 版本管理
- [ ] 每次重新规划版本号递增1（从v1.0开始）
- [ ] 新计划名称 = 原始名称 + " v" + 新版本号
- [ ] 不创建新计划记录，直接覆盖原有计划数据

### 2. 重新规划起点
- [ ] 始终从当前日期的第二天开始
- [ ] 假设当天用户会尽力完成剩余部分

### 3. 内容处理
- [ ] **已完成内容**：不再作为新学习，只纳入复习部分
- [ ] 保留已完成内容的原始学习日期，用于计算复习间隔
- [ ] **未完成内容**：作为新学习内容，从新开始日期按每日新量分配
- [ ] 未完成内容继续使用编号（如从51开始）

### 4. 复习调度
- [ ] 对所有内容（已完成 + 新分配的未完成）按艾宾浩斯曲线重新规划复习
- [ ] 确保复习只在内容被"学习"后触发

### 5. 剩余时间计算
- [ ] **固定从新开始日期到原结束日期的天数**
- [ ] 不基于新计划实际结束日
- [ ] 显示格式："共：X 天（从 YYYY-MM-DD 到原结束日期 YYYY-MM-DD）"

### 6. 边缘情况
- [ ] 如果未完成内容为0：仅重新调度复习，显示纯复习计划
- [ ] 如果当前日期已超过原结束日期：显示"0 天（已逾期）"，但仍生成新计划

### 7. 页面更新
- [ ] 刷新结果表格，显示新计划
- [ ] 更新标题为新计划名称
- [ ] 更新"共多少天"显示
- [ ] 更新"平均每天学习量"显示

### 8. 其他约束
- [ ] 乱序/逆序应用于新计划的复习部分
- [ ] 其他部分不做任何改动

# 待办清单 - 第八轮修复

## 用户反馈的关键问题

### 问题1: 主页背景栏左右白边（已改三次仍未修复）
**现状**：Hero Section背景两侧依旧有白色边框
**要求**：背景必须覆盖整个屏幕宽度，无任何白边
**解决方案**：
- 使用`w-screen`确保宽度覆盖整个视口
- 使用`relative left-1/2 right-1/2 -ml-[50vw] -mr-[50vw]`技巧突破container限制
- 或者将Hero Section移出container外部

### 问题2: 重新规划功能逻辑错误
**现状**：重新规划逻辑不符合用户需求
**要求**：
1. **已学习内容处理**：将当前用户已完成（勾选）的内容都作为接下来的复习部分
2. **开始时间**：从用户未完成计划的第二天开始规划
3. **显示真实时间**：在标题下方的一行小字显示用户真实的时间信息
4. **实时更新**：每日任务要实时更新

**解决方案**：
- 收集所有已完成的单元作为`completedUnits`
- 收集所有未完成的单元作为`uncompletedUnits`
- 重新规划时：
  - 新学习部分：`uncompletedUnits`
  - 复习部分：`completedUnits`（按艾宾浩斯曲线安排复习）
- 开始日期：明天（今天+1天）
- 剩余天数：从明天到原计划结束日期
- 更新计划信息显示真实的时间和任务数

## 注意事项
- 只修改上述两个问题
- 不要修改其他任何功能或样式
- 修改后必须全面测试
